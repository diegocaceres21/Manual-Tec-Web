Nombre Descripcion Usos
Elementos Metadata
meta El elemento meta representa metadata que no puede ser representada por otros elementos relacionados a meta
title Define el titulo del documento que es mostrado en la pestaña de buscador Siempre es usado dentro del elemento head
Elementos Sectioning
article El elemento article representa una composición autocontenida en un documento, pagina, aplicación o sitio la cual pretende ser independientemente distribuida o reusable. Por ejemplo, un post de foro, articulo de revista, etc
  • Cada article debería ser identificado, tipicamente incluyendo un heading como hijo de article
  • Cuando un article está anidado, el elemento interior representa un articulo relacionado a un elemento exterior
  • La informacion del autor de un articulo puede ser proveida mediante el elemento address
  • La fecha y hora de una publicación pueden ser descritas usando el atributo datetime de un elemento time
aside El elemento aside representa una porcion de un documento cuyo contenido está indirectamente relacionado al contenido principal del documento. Frecuentemente se usa para menus laterales o cuadros de dialogo No usarlo para etiquetar texto entre parentesis, ya que estos son considerados parte del flujo principal
nav El elemento nav representa una sección de una página cuyo objetivo es proveer enlaces de navegación, ya sea en el mismo documento o hacia otros
  • No es necesario que todos los links esten contenidos en este elemento. Algunos pueden ser contenidos en el footer
  • Un documento puede tener varios elementos nav. Por ejemplo, uno para navegación por el sitio y otro para navegación solo en la pagina.
  • Los screen readers pueden usar este elemento para determinar si omitir o no la renderización inicial de contenido de solo navegación
section El elemento section representa un sección generica separada de un documento, la cual no tiene un elemento semantico más especifico para representarlo.
  • Solo debería ser utilizado si no hay un elemento más especifico para esto.
  • Cada section debería ser identicado, tipicamente utilizando un elemento heading como hijo del elemento section
Elementos Heading
h1-h6 Representan seis niveles de cabeceras de sección, donde h1 es el nivel más alto y h6 el más bajo
  • Su información puede ser usada por agentes del usuario para construir una tabla de contenido del documento
  • No usar elementos heading para reajustar el tamaño del texto.
  • No saltarse niveles de heading, siempre ir en orden.
  • Evitar usar multiples h1 en una página
Elementos Phrasing
br Es un elemento vacio que produce una linea vacia en el texto Usado para escribir poemas o direcciones
wbr Representa una oportunidad para que el navegador realice un line break Usado para delimitar el texto según la pantalla
sup Especifica texto en linea que será mostrado en el navegador como superindice Usado para exponentes, referencias o numeros ordinarios
sub Especifica texto en linea que será mostrado en el navegador como subindice Usado para formular matematicas, quimicas y notas de pie de linea

Elementos Flow
p Representa un parrafo, es decir bloques de texto separados por espacios vacios entre si. Estos son elementos de bloque.
span Es un contenedor en linea para contenido tipo phrasing, el cual no representa algo hereditariamente. Puede ser usado para agrupar elementos por temas de estilo y solo debe ser usado cuando ningun otro elemento semantico es apropiado
div Es el contenedor generico para contenido tipo flow Solo debe ser usado cuando ningun otro elemento semantico es apropiado
header Representa contenido introductorio, que puede contener elementos heading, pero tambien un logo, un buscador, nombre del autor u otros elementos Puede definir un header global del sitio, descrito como un banner o ser usado conteniendo los elementos heading
footer Representa un pie de pagina para su más cercano ancestro. Generalmente contiene informacion del autor, derechos de autor u otros enlaces. Cuando su ancestro más cercano es el body, aplica un pie de pagina para toda esta. Este no introduce una nueva sección
main Representa el contenido dominante del body en un documento El contenido main debería ser único en el documento y es un elemento estrictamente informativo
hr Representa un linea de break con tematica entre elementos de parrafo Usado para cambios de escena o de topicos
Elementos Sin Categpria
html Representa la raiz de un documento html, por lo que tambien es conocido como el elemento raiz. Todos los otros elementos deben ser descendentes de este
head Contiene informacion leible por la maquina sobre el documento, como su titulo, scripts o estilos
body Representa el contenido de un documento HTML Solo puede haber un body en un documento
